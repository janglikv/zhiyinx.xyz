# 🏪 “知音杂货铺”系统账号与限量库存管理设计方案 (Idea 记录)

这里记录了关于“知音杂货铺”作为系统账号、挂件限量库存领取的核心创意与后续技术演进方向。该想法将为平台后续增加轻量游戏化交互和社交独占属性提供支撑。

---

## 💡 核心创意概述

“知音杂货铺”未来将从一个纯静态/动态的页面挂件，升级为知音平台内一个特殊的**系统级官方账号 (System Account)**。该账号拥有自己专属的装饰物品“库存”，并具有全局唯一性和限量性。

- **官方铺货**：系统以“知音杂货铺”账号的名义，向全局上架一些限量的特色物品挂件（如：特别手绘箭头、金色纪念版便利店等）。
- **玩家领用**：普通注册用户可访问小铺并领走物品挂件。
- **库存清空拦截**：一旦某种物品的全局库存被领光（库存归零），别的玩家将无法再次领取该物品，使其成为平台上的“限量绝版装饰”，激发用户的收藏与互动欲。

---

## 🛠 数据库架构扩展想法 (DB Migrations)

为了支持该模式，我们需要在本地 D1 数据库中建立限量库存与物品扣减之间的强一致性关联。

### 1. 杂货铺商品库存表 `grocery_inventory`
用于配置和维护杂货铺正在售卖/赠送的商品条目：

| 字段名 | 类型 | 约束 | 描述 |
| :--- | :--- | :--- | :--- |
| `item_type` | TEXT | PRIMARY KEY | 物品类型标识 (如 `arrow`, `grocery`, `special_flag`) |
| `display_name` | TEXT | NOT NULL | 物品的展示名称 |
| `stock` | INTEGER | NOT NULL DEFAULT 0 | 剩余可领取的库存数量 |
| `max_stock` | INTEGER | NOT NULL | 初始最大投放量（用于前台展示总量比例） |
| `created_at` | TIMESTAMP| DEFAULT CURRENT_TIMESTAMP | 物品上架投放时间 |

### 2. 用户领用记录表 (复用或扩展 `user_items`)
目前的 `user_items` 表包含了 `id`, `user_id`, `item_type`, `bottom`, `left`，完全可以用来承载领到的挂件。
* **规则设定**：若需限制每位用户只能拥有一个同类限量款，可以在业务代码中拦截，或在数据库加唯一约束：
  ```sql
  UNIQUE(user_id, item_type)
  ```

## 🔌 核心 API 接口规划

为了防止多用户并发抢领导致库存“超卖”（库存变负数），接口需要通过 D1 / SQLite 事务或原子性更新来确保强一致性。

### 1. 获取杂货铺货架列表 `GET /api/grocery/inventory`
- **描述**：拉取目前杂货铺正在提供领取的物品目录和实时库存。
- **返回示例**：
  ```json
  [
    { "item_type": "arrow", "display_name": "手绘大箭头", "stock": 42, "max_stock": 100 },
    { "item_type": "grocery", "display_name": "知音杂货铺", "stock": 0, "max_stock": 10 }
  ]
  ```

### 2. 领取/抢购限量物品 `POST /api/grocery/claim`
- **请求体**：`{ "item_type": "grocery" }`
- **后端执行事务 (Transaction Flow)**：
  1. **检查与锁定**：在事务中执行查询并做库存校验：
     ```sql
     SELECT stock FROM grocery_inventory WHERE item_type = ?;
     ```
  2. **条件拦截**：如果 `stock <= 0`，立即中断，向前端返回错误码 `STOCK_EMPTY` （库存已被抢光）。
  3. **用户排他校验**：校验该 `user_id` 是否已领过同类物品。
  4. **原子扣减库存**：
     ```sql
     UPDATE grocery_inventory SET stock = stock - 1 WHERE item_type = ? AND stock > 0;
     ```
     （`stock > 0` 保证了即使在极高并发下也绝不会产生负库存）。
  5. **发货入库**：向 `user_items` 表中为该玩家新增一条物品记录，并初始化默认像素坐标：
     ```sql
     INSERT INTO user_items (user_id, item_type, bottom, left) VALUES (?, ?, '160px', '30%');
     ```
  6. 提交事务，返回领取成功响应。

## 🎨 前端交互与 UI 展现设想

结合我们目前所拥有的写实画风 SVG，未来的前台交互可以设计得极其生动：

1. **Hover 动态库存面板**：
   - 当玩家鼠标滑过“知音杂货铺”小店时，除了灯光亮起、大门打开、招牌晃动之外，侧边可以淡入一个磨砂质感的小看板，上面显示：
     - 📝 *“手绘大箭头：剩余 42 个”*
     - ⚠️ *“知音小便利店：已售罄 (10/10)”*
2. **写实的“售罄”印章效果**：
   - 点击杂货铺打开领取弹窗时，未领光的物品散发金光供点击；
   - 而已领光的物品会被完全置灰，并且在商品图案上方叠加一个大红色、倾斜盖上的写实印章 —— **“已抢光 (SOLDOUT)”**。
3. **通知广播系统**：
   - 当有玩家成功领走最后一个限量款物品时，底部 `StartBar` 上方可以弹出一个微弱的系统广播提示：“用户 *xxx* 刚刚领走了最后一个【知音杂货铺】限量挂件！”。
